## 📖 專案背景與醫療現場痛點

手術室（OR）是醫療機構中運行成本最高、資源最密集的單位之一。傳統的手術排程多依賴「醫師個人經驗」或「歷史平均值」進行估算，往往忽略了病患個體差異與手術複雜度，容易導致以下代價：
* **時間低估（超時）：** 引發後續手術連帶延誤、醫護人員超時加班與過勞，甚至被迫取消當日其他手術。
* **時間高估（閒置）：** 導致手術室資源空置、醫療服務量能浪費，降低醫院整體營運效益。

本專案利用**機器學習（Machine Learning）**技術取代傳統經驗法則，針對不同臨床科別建立動態預測模型，協助優化手術室排程效率，在「避免醫護過勞」與「提升資源利用率」之間取得最佳平衡。

## 🛠️ 特徵工程與資料預處理管線 (Data Pipeline)

專案導入 `Pandas` 與 `NumPy` 建立自動化資料預處理管線，從結構化醫療數據中清洗並提取出 **11 個核心特徵欄位** 作為模型輸入：

| 特徵名稱 | 臨床維度解構 | 預處理轉換方式 |
| :--- | :--- | :--- |
| **年齡(G)、性別(H)** | 病患基本生理特徵與個體差異 | 數值填補與標籤編碼 (Label Encoding) |
| **身份(I)、分類(J)** | 病患就醫類別與案件屬性差異 | 類別資料轉換、缺失值自動封裝為 'Unknown' |
| **麻醉(K)、手術名稱(L)** | 手術複雜度與醫療干預手段 | 複雜文本多分類標籤編碼 (Label Encoder) |
| **主治醫師(AF)、醫師年資(AG)** | 執刀醫師之臨床經驗與技術熟練度 | 年資數值標準化、醫師 ID 編碼化 |
| **手術數量(AQ)、醫師人數(BG)** | 醫療團隊配置與當日手術排程負荷密度 | 團隊規模與歷史負荷特徵量化 |
| **分類(AZ)** | 手術特定的衍生子分類特徵 | 臨床情境特徵矩陣整合 |

* **目標變數 (Target Variable)：** 手術實際耗時（分鐘）`手術時間（分）(BQ)`。

## 🤖 模型架構與多科別動態調優策略

預測核心採用 **隨機森林迴歸模型 (Random Forest Regressor)**，因其具備極佳的抗雜訊能力、不易過擬合，且能輸出特徵重要性以供臨床解讀。

為了適應不同醫學科別（Departments）間的手術特性差異，本研究採取**「分流管理、動態調優」**策略，針對 5 大核心科別進行客製化超參數控制：

* **核心基準參數：** `n_estimators=500`（維持穩定預測能力）、`min_samples_leaf=5`（防止極端值干擾）、`random_state=42`。
* **動態深度控制 (Max Depth Strategy)：**
  * **複雜型科別 (ENT 耳鼻喉科、GS 一般外科、GU 泌尿外科)：** 手術時間受併發症與技術影響變異極大，模型最大深度調整為 `max_depth=30`，以捕捉高維度非線性特徵。
  * **常態型科別 (OPH 眼科、ORTH 骨科)：** 手術流程相對標準化，模型最大深度限制為 `max_depth=20`，避免模型過度擬合雜訊。

## 📈 最佳化歷程與實驗成效摘要

模型開發階段採取嚴謹的 **80/20 驗證機制（Train/Val Split）** 模擬真實盲測。在歷經 6 次數據試算與調優的過程中，本團隊成功克服了多個臨床數據特有的雜訊挑戰：

### 1. 關鍵優化案例：ENT（耳鼻喉科）過度擬合修正
* **問題：** 在第 4 次試算中嘗試對 ENT 科別採全數據保留，導致模型對異常手術樣本過度擬合，平均絕對誤差（MAE）飆升至 `42.51`。
* **對策：** 第 5 次優化果斷引入分流策略，實施 `0.97%` 的數據雜訊過濾與微調。
* **成效：** 成功將預測誤差壓回 **`34.93`**，顯著提升了泛化能力。

### 最終實驗成效
經指導教師與自測集嚴格交叉驗證，5 大科別模型的平均絕對誤差（MAE）表現優異（如 GS 一般外科 MAE 達 `39.03`、ORTH 骨科 MAE 達 `31.19`、OPH 眼科 MAE 達 `13.32`），**全數通過醫療排程之模型效能考核指標**。
